## 📋 需求描述

设计萨摩耶情侣聊天应用的管理后台整体架构。

### 业务背景
- 需要一个管理后台来管理用户、内容和查看数据
- 支持管理员登录和权限控制
- 需要与现有聊天应用共享用户数据库

### 功能范围
1. **管理员系统**
   - 管理员登录/登出
   - 权限管理（超级管理员/普通管理员）
   - 操作日志记录

2. **用户管理**
   - 用户列表（分页、搜索）
   - 用户详情查看
   - 禁用/启用用户
   - 用户数据统计

3. **内容审核**
   - 消息列表查看
   - 违规内容删除
   - 用户警告/封禁

4. **数据统计**
   - 用户增长曲线
   - 活跃度统计
   - 消息量统计
   - 配对成功率

## 🛠️ 技术约束

### 必须使用
- Node.js (与现有后端一致)
- SQLite (与现有数据库一致)
- JWT 认证

### 推荐考虑
- React/Vue (管理后台前端)
- Ant Design/Element UI (UI 组件库)
- Express/Koa (后端框架)

### 性能要求
- 页面加载 < 3 秒
- API 响应 < 500ms
- 支持 100+ 并发管理员

### 安全要求
- HTTPS 加密
- 密码加密存储
- 操作日志记录
- 会话超时
